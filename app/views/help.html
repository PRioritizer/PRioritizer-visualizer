Icon Description
Mergable. The pull request can be merged directly.
Conflicted. The pull request cannot be merged without resolving conflicts.
Importance indicator. Based on previous actions on past pull requests, a prediction is made whether the pull request requires action. This icon is only visible on the pull requests with the highest chance of requiring an action. The maximum number of pull requests with this icon is five.
Project member. The user that created the pull request is a project member.
Non-member. The user that created the pull request is not a project member.
Contributed. The percentage next to this icon shows the fraction of commits in the project that are already authored by requester. This can indicate how involved the requester is.
Accept ratio. The percentage next to this icon shows the fraction of pull requests created by the requester that are accepted. This can give an indication of the reliability of the requester.
Pairwise conflicts. The number next to this icon shows the amount of conflicts between the pull request and other pull requests targeting the same branch. Knowing this, you can decide to merge heavily conflicted pull request at a later stage to concentrate multiple conflicts in one merge action.
Comments. The number next to this icon shows the (estimated) sum of review and issue comments.
Files changed. The number next to this icon shows the number of changed files in the pull request.
Commits. The number next to this icon shows the number of commits in the pull request.
  Code churn. These icons show the proportion of lines added (green) and deleted (red).
